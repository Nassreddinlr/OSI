How It Works

Every step, explained

The user runs one file. No terminal knowledge needed, no config editing, no reading Dortania guides at 2am. Here's what happens under the hood.

🔍
Step 1 — Deep Hardware Scan
The tool queries every hardware component using OS-native APIs. On Windows it uses WMI. On Linux it reads lspci, dmidecode, /proc, and /sys. It builds a complete hardware_profile.json that drives every decision downstream. Nothing is guessed — if it can't detect something, it flags it and asks the user.
⚙️
Step 2 — EFI Config Generation
Using the hardware profile, the tool selects the right OpenCore version, picks every required kext from the database, generates a valid config.plist from templates, applies CPU quirks, sets the SMBIOS to the right Mac model, and generates SSDTs for ACPI patches. All automated via a compatibility matrix JSON.
📥
Step 3 — Download Everything
macOS recovery is fetched directly from Apple's CDN (same as macrecovery.py). OpenCore and all kexts are pulled from their official GitHub Releases. Integrity is verified with checksums. User picks macOS version: Ventura, Sonoma, or Sequoia.
💾
Step 4 — USB Creation
The USB is formatted, the recovery image written, the EFI partition mounted, and the generated EFI folder copied in. On Windows: diskpart + mountvol. On Linux: dd + parted + mount. The result is an identical bootable USB on both platforms.
✅
Step 5 — Validation & Reporting
Before the user touches their BIOS, the tool verifies the USB: EFI partition exists, config.plist is valid XML, all kexts are in place, OpenCore is present. It generates a short summary report the user can save — useful for troubleshooting if boot fails.
🍎
Step 6 — Boot & Install
User plugs in USB, restarts, enters BIOS boot menu, selects USB. OpenCore bootloader appears. macOS installer loads. User formats the target drive and installs. Post-install, the tool can optionally help mount and copy the EFI from USB to the internal drive's EFI partition.

Risk Register

What can go wrong

Every risk has a mitigation baked into the build plan. No surprises.

RiskSeverityMitigation
config.plist wrong value → machine won't bootHIGHTest every template on real hardware. Start with 10 most common boards only. Ship OC debug mode enabled by default so boot logs are visible.
AMD kernel patches differ per Zen genHIGHVersioned patch table per Zen 2/3/4. Detect Ryzen generation precisely from CPU model string. Test each separately before merging.
Linux: dd writes to wrong disk (data loss)HIGHShow device name + size + label. Require user to type device name to confirm. Never auto-select. Add "are you sure?" dialog with red warning text.
Intel I225/I226 Ethernet quirksHIGHKnown fix: apply e1000LegacySupport patch in config.plist. Detect I225/I226 by PCI ID and auto-apply.
USB port count exceeds macOS limit of 15HIGHGenerate a USBMap.kext for common board models. For unknown boards, enable XhciPortLimit quirk with clear warning that a proper USB map is needed post-install.
Apple changes macOS recovery CDN URLsMEDMirror macrecovery.py URL logic. Add self-update for URL list via GitHub Gist. Alert user if download fails.
OpenCore release breaks existing configsMEDPin OC version per macOS version in compat_matrix.json. Don't auto-update OC; require explicit version bump in matrix.
Non-standard ACPI on exotic motherboardsMEDScope to Z490/Z590/Z690/Z790/B650/X570/B550 only in v1. Flag unsupported boards clearly with a link to Dortania guide.
Windows AV flags the .exe as malwareMEDCode-sign the binary. Publish full source on GitHub. Build community reputation. Submit to AV vendors for whitelisting.
iMessage/FaceTime requires valid SMBIOSMEDGenerate unique MLB + serial + ROM using GenSMBIOS logic. Warn user these values are unique and must not be shared. Document post-install iServices setup.
Linux: CustomTkinter broken on WaylandMEDDetect XDG_SESSION_TYPE. Set DISPLAY=:0 for Xwayland fallback. Add CLI-only mode as fallback if GUI fails.
Sleep/wake doesn't work post-installMEDOut of scope for tool — document known issues per platform. Generate SSDT-GPRW or SSDT-PTXH for common boards.
CFG Lock not unlockable on some boardsMEDDetect Intel CPU + apply AppleCpuPmCfgLock and AppleXcpmCfgLock quirks as fallback. Document how to unlock via BIOS mod.
NVMe SSD not recognized (some Samsung/WD)LOWKnown models: detect by PCI vendor ID and auto-apply SetApfsTrimTimeout or NVMeFix.kext. Document known-bad models.
USB write fails (bad USB drive)LOWVerify write with MD5 checksum. Clear error message. Recommend USB 3.0 drive, 16GB+. Retry option.
Linux: python3-tk not installedLOWDetect at startup. Print exact install command and exit gracefully. sudo apt install python3-tk.
